Requisitos No Funcionales

    El sistema debe soportar hasta 10 millones de usuarios activos concurrentes sin degradación significativa del rendimiento.
    Las publicaciones de texto deben tener un límite de 5,000 caracteres.
    La red social debe estar disponible 24/7 con un tiempo de inactividad planificado no mayor a 2 horas al mes para mantenimiento.
    La plataforma debe ser accesible desde los navegadores Chrome, Mozilla Firefox, Safari y Edge, tanto en versiones de escritorio como móviles.
    Las imágenes subidas deben comprimirse automáticamente para no exceder los 5 MB de tamaño sin perder calidad notable.
    La aplicación móvil debe ser compatible con dispositivos que utilicen Android 8+ e iOS 14+.
    Las consultas a la base de datos deben tener un tiempo de respuesta máximo de 200 ms para el 95% de las operaciones.
    Las contraseñas de los usuarios deben estar cifradas usando algoritmo bcrypt con un mínimo de 12 rondas de hashing.
    El sistema debe ser capaz de procesar 1 millón de solicitudes por minuto en picos de tráfico.
    La aplicación debe cumplir con las normas de accesibilidad WCAG 2.1 Nivel AA para garantizar su uso por personas con discapacidades.
    La API REST pública debe estar protegida mediante OAuth 2.0 y solo accesible a aplicaciones autorizadas.
    Las publicaciones y comentarios eliminados deben almacenarse en un estado "borrado" durante 30 días antes de su eliminación permanente.
    Las transferencias de datos entre cliente y servidor deben realizarse mediante protocolo HTTPS con cifrado TLS 1.2 o superior.
    El sistema debe soportar la carga simultánea de videos de hasta 2 GB con un tiempo máximo de procesamiento de 5 minutos.
    Los usuarios deben poder recibir notificaciones en tiempo real con una latencia máxima de 1 segundo.
    El nombre de usuario debe tener un límite de 15 caracteres.
    La aplicación deberá poder ser visualizada en navegadores Chrome, Mozilla Firefox y Safari.
